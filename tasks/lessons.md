# Lessons Learned

## 2026-02-27
- 当用户要求“引用 raw url”时，不应停留在本地相对路径，需先读取仓库 `origin` 与当前分支，再生成精确的 GitHub Raw 地址。
- 对“先对比、再覆盖”的需求，必须先做内容哈希或 diff 验证，再执行覆盖，避免误覆盖。
- 在 Surge 模块改 `script-path` 后，需立即逐行校验所有目标条目，避免只改一处导致行为不一致。

## 2026-03-12
- 当脚本参数名存在历史写法差异（如 `reset_day` 与 `resetday`）且用户用新写法描述需求时，应优先做兼容处理并明确优先级，避免只支持单一命名导致旧配置失效。
- 修改 Surge 订阅脚本的参数来源时，先核对现网配置里已使用的参数名；若用户描述与现状不一致，应优先做兼容而不是直接替换旧键名。
